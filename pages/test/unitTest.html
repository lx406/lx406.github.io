# 单元测试原则

# AIR原则

  1. A：Automatic（自动化） 不能有对人工操作的依赖
  2. I：Independent （独立性） 用例直接不能有逻辑、顺序上的依赖关系
  3. R：Repeatable （可重复） 尽可能不依赖外部环境

# BCDE原则

  1. B：Border 边界值测试：循环边界、特殊取值、特殊长度、数据顺序等
  2. C：Correct 正确的输入，并得到预期的结果
  3. D：Design 与设计文档相结合，不要对着代码编写单元测试
  4. E：Error 强制错误信息输入（如：非法数据），并得到预期的结果

# 单元测试工具

  1. react ——> Enzyme
  2. vue ——> @vue/test-utils

# Enzyme

Airbnb出品，是对react官方测试工具库的封装

下面是三个核心api：

  1. Shallow Rendering (shallow) 浅渲染，将组件渲染成虚拟VDOM，不渲染所有子组件
  2. Full Rendering (mount) 全渲染，将react组件渲染成真是DOM节点
  3. Static Rendering (render) 静态渲染，将react组件渲染成静态的HTML字符串
