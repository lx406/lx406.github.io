# 浏览器工作原理

当我们打开浏览器，观察任务管理器时，可以看到浏览器当前有4个进程，分别是主进程、网络进程、GPU进程、渲染进程。那么浏览器为什么要分成这么多进程呢，我们先从进程与线程说起。

# 进程与线程

# 进程

运行程序时，操作系统会为该程序创建一块内存，用来存放代码、运行中的数据以及一个执行任务的主线程，这样的运行环境叫做进程

# 线程

线程依附于进程，多线程可并行处理提高运算效率。

# 两者间特点

  1. 进程中任一线程出错，都会导致整个进程崩溃
  2. 线程之间共享进程数据
  3. 进程关闭后，操作系统会回收进程内存
  4. 进程间相互隔离

# 浏览器进化史

# 单线程浏览器

所有功能都运行在一个进程里。

这样会导致浏览器不稳定、不流畅、不安全

  1. 不稳定：插件以及渲染引擎都不稳定，一旦崩溃整个浏览器都会崩溃
  2. 不流畅：javascript执行环境也都在一起，如何某些代码影响了执行效率，整个浏览器都变得卡顿
  3. 不安全：运行插件时获取系统权限，释放病毒、窃取账号密码等

# 多线程浏览器

分成浏览器进程、渲染进程、网络进程、GPU进程、插件进程

这样可以很好地避免了不稳定、不流畅、不安全的问题

但出现了两个问题：

  1. 更高的资源占用：每个进程都会包含公共基础结构副本
  2. 更复杂的体系架构

# 面向服务架构SOA

将公共基础结构分成多个服务，每个服务在独立进程里运行，通过协议接口等方式进行通信，进行解耦。
